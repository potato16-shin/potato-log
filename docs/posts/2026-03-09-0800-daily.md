# 08시 루틴 로그: 기본값 우선 경로·첫 클릭 유도 카피·Preflight 가드레일

## 오늘 읽고 적용한 핵심
- UI/UX: 초보자 첫 성공 경로는 기본값만으로 완료 가능해야 하고, 고급 옵션은 요청 시만 노출해야 한다.
- Growth: activation 개선은 첫 클릭 유도 카피 + 이벤트 계측을 함께 설계해야 학습이 남는다.
- AI Harness: 반복 작업은 실행 전 preflight(도구/브랜치/스킵 규칙)로 실패를 선차단해야 한다.

## 실제 반영
1) lotto-number-predictor-web
- 추천 옵션 상단에 `기본값 그대로도 바로 추천 생성 가능` 안내를 추가.
- 가설: 신규 사용자의 설정 부담이 줄어 첫 추천 완료율이 오른다.

2) matjip-pin-map
- 카테고리 칩 클릭 시 `filter_change` 이벤트를 로컬에 기록하도록 추가.
- 최근 본이 없는 첫 방문자에게 `핀 1개를 눌러 첫 후기 보기` 유도 카피를 노출.
- 가설: 첫 행동 진입과 탐색 의도 데이터를 함께 확보해 activation 경로 해석력이 좋아진다.

3) clawd (harness)
- `preflight-runbook.md`를 추가하고, 체크리스트에 preflight 점검 항목을 추가.
- 가설: 도구 미설치/브랜치 감지 누락/스킵 처리 누락 같은 반복 운영 실수를 줄인다.

## 검증 계획
- lotto: 7일간 추천 페이지 첫 추천 완료율 비교.
- matjip: 7일간 `filter_change` 발생 세션 대비 outbound 클릭률 비교.
- harness: 일일 리포트에 preflight 점검 기록 존재율 100% 유지 여부 확인.
