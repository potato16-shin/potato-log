## 오늘 읽은 것

### 1) NN/g — Error-Message Guidelines
- 오류는 발생 지점 가까이에 보여줘야 복구 속도가 빨라진다.
- 기술 디테일보다 사용자가 지금 무엇을 할 수 있는지 먼저 알려줘야 한다.
- 차단 오류는 최소화하고, 차단 시 즉시 행동 버튼을 제공해야 한다.
- 원문: <https://www.nngroup.com/articles/error-message-guidelines/>

### 2) Martin Fowler — Feature Toggles
- feature flag는 빠른 배포와 안정적 롤백을 동시에 가능하게 하는 운영 패턴이다.
- 토글은 owner/만료/제거 기준까지 포함해 설계해야 부채가 쌓이지 않는다.
- 사용자 영향 변경은 기본 OFF, 점진 ON 전략이 안전하다.
- 원문: <https://martinfowler.com/articles/feature-toggles.html>

### 3) Google SRE Book — Monitoring Distributed Systems
- page는 사용자 영향이 크고 즉시 조치 가능한 사건에 한정해야 한다.
- 운영 대시보드는 공통 판단을 위해 지연/트래픽/오류/포화 신호를 먼저 보여야 한다.
- 모니터링은 관측이 아니라 복구와 재발 방지로 연결되어야 한다.
- 원문: <https://sre.google/sre-book/monitoring-distributed-systems/>

## 오늘의 적용 액션
- 오류 문구를 "영향 + 다음 행동" 템플릿으로 통일
- 사용자 영향 변경은 kill metric/rollback 경로를 사전 정의
- page 승격 기준을 증상 기반 + 즉시 실행 가능 액션 기준으로 강화

## 오늘 반영한 UI/UX 개선

### 1) lotto-number-predictor-web
- 적용 원칙: **Error copy must translate technical cause into user action**
- 변경 파일: `src/app/history/page.tsx`
- 변경 내용: 삭제 실패 문구를 단순 실패에서 "영향 + 다음 행동" 문장으로 개선
- 기대 효과: 실패 후 재시도율 증가, 불필요한 이탈 감소
- 검증 체크:
  1) 삭제 실패 시 원인 추측 없이 즉시 재시도 행동이 보이는지
  2) 문구가 한 문장으로 이해되는지

### 2) matjip-pin-map
- 적용 원칙: **Error copy must translate technical cause into user action**
- 변경 파일: `src/components/useKakaoLoader.ts`
- 변경 내용: SDK/환경 오류 문구를 기술 용어 중심 표현에서 사용자 행동 중심 한국어 문구로 전환
- 기대 효과: 지도 로드 실패 시 재시도 행동 명확화
- 검증 체크:

  2) 내부 구현 용어 없이도 상황 이해가 가능한지
