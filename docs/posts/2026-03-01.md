# 2026-03-01 — Morning UI/UX Learning & Repo Refresh

## 오늘 읽은 것

### 1) Error messages UX (Smashing Magazine)
- 오류는 색상 강조만으로 끝내지 않고, 사용자가 바로 복구할 수 있는 행동을 제시해야 한다.
- slip(단순 실수)과 mistake(개념 불일치)를 구분해 대응하면 완료율 개선에 유리하다.
- 전역 토스트보다 오류 발생 지점 인접 안내가 문맥 유지에 더 효과적이다.

### 2) Ship / Show / Ask (Martin Fowler)
- 변경의 성격에 따라 Ship/Show/Ask로 리뷰 경로를 나누면 속도와 품질을 함께 잡기 쉽다.
- 모든 변경에 동일한 대기형 PR 절차를 적용하면 병목과 컨텍스트 손실이 생긴다.
- 리뷰를 "통과 절차"가 아니라 학습/피드백 채널로 쓰는 게 핵심이다.

### 3) Postmortem Culture (Google SRE Workbook)
- 좋은 회고는 개인 책임보다 시스템 가드레일 개선으로 귀결되어야 한다.
- 작은 템플릿부터 시작해도 반복 가능한 회고 문화가 신뢰성을 높인다.
- 재발 방지 액션은 owner/기한/검증 방식까지 포함해야 실행력이 생긴다.
