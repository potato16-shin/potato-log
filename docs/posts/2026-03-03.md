# 2026-03-03 — Morning UI/UX Learning + Service Apply

## 오늘 읽은 것 (3)

### 1) Progressive Disclosure (NN/g)
- 핵심 3줄
  - 처음 화면에는 핵심 작업에 필요한 옵션만 보여준다.
  - 고급 옵션은 요청 시점에만 드러내면 학습 비용과 오류가 줄어든다.
  - 보여주는 정보 자체가 우선순위 신호가 된다.
- 바로 적용
  - 로또 추천 화면의 고급 옵션(기준기간/추천방식/홀짝)을 접기 섹션으로 분리.

### 2) Ship / Show / Ask (Martin Fowler)
- 핵심 3줄
  - 변경마다 동일한 검토 흐름을 강제하면 PR 체증이 생긴다.
  - Ship/Show/Ask를 먼저 정하면 속도와 품질 균형이 좋아진다.
  - Show는 빠른 병합과 팀 학습을 동시에 만든다.
- 바로 적용
  - 루틴 리포트에서 변경 성격에 맞는 검토 경로를 명시하도록 유지.

### 3) Monitoring Distributed Systems (Google SRE)
- 핵심 3줄
  - 사람을 깨우는 알림은 사용자 영향이 명확할 때만 쓴다.
  - 알림에는 즉시 실행할 초기 조치가 있어야 한다.
  - 모니터링은 골든 시그널 기반으로 시작하는 것이 안정적이다.
- 바로 적용
  - 오류 문구를 사용자 영향 + 즉시 행동(재시도) 형태로 통일.
