# 2026-03-02 — Morning UI/UX Learning & Repo Refresh

## 오늘 읽은 것

### 1) Progressive Disclosure (Nielsen Norman Group)
- 고급 옵션은 기본 화면에서 숨기고, 사용자가 요청할 때만 노출해야 학습 부담과 실수를 줄일 수 있다.
- 첫 화면에 무엇을 남길지가 곧 "중요도" 신호가 된다.
- 웹/모바일 모두 정보 과밀 화면에서 효과가 크다.

### 2) Idempotency for Robust APIs (Stripe)
- 네트워크 실패 상황에서는 성공/실패가 애매해 재시도가 기본 전략이 된다.
- 안전한 재시도를 위해 쓰기 API는 idempotency 계약이 필요하다.
- 사용자 안내 문구도 "중복 없이 재시도 가능"을 명확히 전달해야 한다.

### 3) Ship / Show / Ask (Martin Fowler)
- 변경마다 Ship/Show/Ask를 먼저 선택하면 리뷰 병목과 무검토 리스크를 동시에 줄일 수 있다.
- Show는 빠른 머지와 피드백 확보를 같이 달성하는 방식이다.
- Ask는 고위험 변경에 집중하는 게 효과적이다.
