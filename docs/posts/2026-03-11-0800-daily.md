# 08시 루틴 로그: 고급옵션 단계화·루프 입력 태깅·eval 기준 명시

## 오늘 읽은 것 (핵심 3줄 + 적용)

### 1) Progressive Disclosure (NN/g)
- 기본 경로의 핵심 액션은 첫 화면에 남기고, 고급 옵션은 요청 시에만 여는 게 유리하다.
- 복잡도를 없애는 것보다 노출 순서를 설계하는 게 더 현실적이다.
- 적용: 디자인 플레이북에 `기본 경로 상시 노출 + 고급 옵션 on-demand` 원칙 추가.
- 링크: https://www.nngroup.com/articles/progressive-disclosure/

### 2) Error Message Guidelines (NN/g)
- 오류는 발생 지점 근처에 보여줘야 사용자가 즉시 원인을 이해한다.
- 복구 행동(재시도/수정) 없는 오류 문구는 이탈을 만든다.
- 적용: 서비스 오류/빈 상태 안내는 "문제 + 다음 행동"을 같이 제공하는 원칙으로 통일.
- 링크: https://www.nngroup.com/articles/error-message-guidelines/

### 3) Activation Metrics (PostHog)
- activation은 단일 이벤트가 아니라 retention 상관 행동 조합으로 정의해야 한다.
- 3~5개 이벤트 번들을 후보로 두고 retention uplift로 비교해야 한다.
- 적용: growth playbook에 activation 후보를 조합 단위로 실험하는 원칙 추가.
- 링크: https://posthog.com/product-engineers/activation-metrics

### 4) Growth Loops (Reforge)
- 퍼널만 보면 공유/재열기 같은 루프 입력 최적화가 빠진다.
- 루프 입력은 source 태깅이 있어야 기여 분석이 가능하다.
- 적용: matjip 외부이동 UTM campaign을 source별로 분리.
- 링크: https://www.reforge.com/blog/growth-loops

### 5) Working with evals (OpenAI)
- eval은 입력 스키마와 채점 기준(testing criteria)을 명시해야 재현 가능하다.
- 변경 전/후를 같은 기준으로 비교해야 회귀를 잡을 수 있다.
- 적용: AI Harness playbook에 `eval 기준 명시` 검증 원칙 추가.
- 링크: https://developers.openai.com/api/docs/guides/evals

## 적용 변경
- **lotto-number-predictor-web**: 공유 섹션 카피를 전환 측정 중심으로 개선하고, waitlist UTM campaign을 `first_value_share`로 통일.
- **matjip-pin-map**: 외부 이동 링크 UTM campaign을 `pin_to_blog_<source>`로 분리해 source 기여 분석 정밀도 강화.
- **clawd 하네스**: AGENTS에 레포 최신화 보고 필수 필드(`current/default/dirty`) 규칙 추가.
