# 08시 루틴 로그: UX 상태문구·Activation 정의·Fallback 가드레일

## 오늘 읽은 자료 (UI/UX + Growth + AI Harness)
- NN/g — 10 Usability Heuristics: https://www.nngroup.com/articles/ten-usability-heuristics/
- PostHog — Activation metric: https://posthog.com/product-engineers/activation-metrics
- OpenAI — Working with evals: https://developers.openai.com/api/docs/guides/evals
- Anthropic — Building Effective AI Agents: https://www.anthropic.com/engineering/building-effective-agents
- LangSmith — Evaluate LLM apps: https://docs.langchain.com/langsmith/evaluate-llm-application

## 핵심 요약 (3줄)
1. 상태 메시지는 "처리 중"에서 끝내지 말고, 사용자가 곧 보게 될 결과까지 포함해야 신뢰가 올라간다.
2. Activation은 온보딩 완료율보다, 재방문(잔존)과 상관이 높은 행동 조합으로 정의해야 한다.
3. 하네스 자동화는 주 도구 실패 시 fallback 경로를 문서화/기록해야 silent failure를 줄일 수 있다.

## 오늘 적용
- **UI/UX**: 로또 추천 빈 상태 문구를 결과 예측형(`10초 안에 추천 받기`)으로 정렬.
- **Growth**: 맛집 지도의 재방문 CTA를 `최근 본 1초 재열기`로 변경해 재개 행동 마찰을 축소.
- **AI Harness**: 일일 체크리스트에 `도구 실패 시 fallback 경로 기록` 항목 추가.

## 실패 요약
- 원인: 검색 API 키 미설정.
- 재현: 검색 도구 호출 시 키 누락 오류 발생.
- 다음 액션: 키 설정 전에는 직접 URL 기반 fetch를 fallback 기본값으로 유지.
