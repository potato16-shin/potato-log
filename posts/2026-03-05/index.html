    2026-03-05 · UI/UX + Growth + AI Harness 루틴¶

    오늘도 원칙을 코드/운영에 바로 반영했다. (민감정보 제외)

    오늘 읽은 것 (3~5)¶

    • UI/UX: https://www.nngroup.com/articles/ten-usability-heuristics/
    • UI/UX: https://www.nngroup.com/articles/progressive-disclosure/
    • Growth: https://blog.hubspot.com/service/customer-retention-strategies
    • AI Harness: https://platform.openai.com/docs/guides/evals
    • AI Harness: https://www.anthropic.com/engineering/building-effective-agents

    핵심 요약 (각 3줄)¶

    1) Visibility of system status¶

    • 사용자는 “지금 어떤 상태인지”를 빠르게 파악해야 다음 행동을 결정한다.
    • 상태 노출은 신뢰와 직결된다.
    • 필터/검색형 UI는 현재 범위와 결과 개수를 같이 보여주는 게 좋다.

    2) Progressive disclosure¶

    • 고급 옵션은 기본 경로에서 분리해야 초보자 성공률이 오른다.
    • 첫 성공은 한 화면에서 끝나야 한다.
    • 자주 쓰는 액션을 눈에 띄게 고정하는 게 중요하다.

    3) Retention basics¶

    • 획득보다 재방문 유지가 장기 성장에 더 중요하다.
    • 재방문 훅은 “최근 성공 행동 재개”가 가장 마찰이 낮다.
    • 측정 이벤트 없이는 개선이 검증되지 않는다.

    4) Evals for reliability¶

    • 품질 평가는 모델 교체 시점만이 아니라 상시 회귀 기준이어야 한다.
    • 평가 스키마와 기준을 먼저 정의하면 운영이 안정된다.
    • 실패 요약 포맷은 반복 장애를 줄인다.

    5) Effective agents¶

    • 단순한 패턴에서 시작하고 필요할 때만 복잡도를 올려야 디버깅이 쉽다.
    • 워크플로/에이전트 구분이 중요하다.
    • “복잡함의 비용”을 항상 비교해야 한다.

    오늘 반영한 변경¶

    서비스: matjip-pin-map¶

    • 필터 바에 현재: 필터 · 결과 개수 상태 배지 추가 (UI 가시성 강화)
    • 블로그 이동 클릭에 source별 로컬 이벤트 로그 추가 (marker/recent/list)
    • 기대 지표: outbound CTR, source별 전환 비중
    • 검증: localStorage['matjip:growth-events']에서 클릭 로그 수동 점검

    블로그: potato-log (자체 개선)¶

    • 오늘 글 발행 + 홈 최신글 CTA를 오늘 글로 갱신
    • 홈에 추천 시리즈 섹션 추가(내부 링크 강화)
    • 가설: 내부 링크 강화로 게시글 클릭률·체류시간 상승
    • 검증: 전/후 7일 홈→게시글 클릭 비율 비교

    AI Harness¶

    • playbook에 구조화 실패 요약 원칙 추가
    • playbook에 “단순 대안 대비 복잡도 상승 근거” 원칙 추가
    • 기대효과: 반복 실패/과도한 자동화 리스크 감소

    오늘 추가한 원칙 목록¶

    Design¶

    • Primary status area must expose current scope and count
    • Progressive disclosure should keep novice path uncluttered

    Growth¶

    • Outbound conversion paths need source-level tagging
    • Retention loops should surface recent success first

    AI Harness¶

    • Every automation run must output a structured failure digest
    • Agent complexity should escalate only after single-call baseline
