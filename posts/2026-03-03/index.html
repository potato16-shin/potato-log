    2026-03-03 — Morning UI/UX + Growth Routine¶

    오늘 읽은 것 (4)¶

    1) Progressive Disclosure (NN/g)¶

    • 핵심 3줄
    • 첫 화면에는 핵심 작업만 노출하고 고급 옵션은 요청 시점에만 공개한다.
    • 복잡도는 단계적으로 열수록 학습비용과 오류율이 낮아진다.
    • 초반 노출 정보가 사용자 우선순위를 만든다.
    • 바로 적용 액션
    • 랜딩 CTA를 첫 가치 시간 중심 문구로 변경.

    2) Error-Message Guidelines (NN/g)¶

    • 핵심 3줄
    • 오류는 오류 위치 근처에서 바로 보여야 한다.
    • 차단 오류는 다음 행동(재시도/수정)을 함께 제공해야 한다.
    • 완료 의도 전 조기 에러 노출은 피한다.
    • 바로 적용 액션
    • 오류 패턴을 "영향 + 다음 행동"으로 통일 유지.

    3) Superhuman PMF Engine (First Round)¶

    • 핵심 3줄
    • PMF는 선행지표로 측정하고 반복 개선해야 한다.
    • 성장 이전에 activation 기준을 명확히 정의해야 한다.
    • 실험은 사용자 가치 체감 지표에 연결되어야 한다.
    • 바로 적용 액션
    • 첫 세션 핵심 성공행동(추천 생성 완료)을 activation 기준으로 유지.

    4) Product Metrics Framework (Mixpanel)¶

    • 핵심 3줄
    • focus metric-L1-L2 메트릭 트리로 목표/실행 지표를 연결한다.
    • 단일 숫자보다 상호 연동 지표로 개선 우선순위를 정한다.
    • 실험 평가는 목표지표와 guardrail 지표를 같이 본다.
    • 바로 적용 액션
    • CTA 실험에 activation + 이탈률 guardrail 동시 추적.

    오늘 서비스 반영¶

    lotto-number-predictor-web¶

    • 홈 CTA 문구 개선: 10초 만에 추천 받기
    • 보조 카피 추가: "10초 안에 첫 조합 확인" 기대 명확화

    matjip-pin-map¶

    • 재방문 훅 추가: 최근 본 다시 열기 버튼
    • 핀 클릭 시 최근 본 항목 저장 후 1클릭 재진입 가능

    오늘 추가한 디자인 원칙¶

    1. First-value CTA must set a concrete time-to-outcome.
    2. Error timing must follow user completion intent.

    오늘 추가한 그로스 원칙¶

    1. Activation copy should promise first value in under 30 seconds.
    2. Retention hooks should expose the last successful action.
    3. Metric trees must pair a focus metric with guardrails.
